# 设计系统说明

## 设计目标
这是一个“历史解释型专题站”，不是论文网页化，也不是刑侦故事页。视觉应当克制、安静、有阅读秩序，强调史料感与解释力。

## 选定设计方向
采用 **Notion × Claude 的混合设计语言**：
- **Notion**：负责信息架构、白底阅读秩序、极轻边框、低噪音模块化
- **Claude**：负责温暖纸面感、历史专题气质、略带编辑感的标题氛围
- **Mintlify**：仅借鉴其文档站式清晰导航与多页面阅读效率，不直接照搬其偏技术化氛围

## 视觉原则
1. 白底与暖米白交替，不用纯黑大块压迫
2. 标题有编辑感，正文保持高可读性
3. 不用浓重革命红黑视觉，不做情绪化历史叙事
4. 所有强调色应服务于导航、卡片层级与时间线，而不是制造视觉噪音

## 色彩建议
- 背景主色：#f7f4ee
- 内容底色：#ffffff
- 主文字：#1d1c19
- 次文字：#5f5a54
- 细边框：rgba(0,0,0,0.08)
- 强调色：#c46a4a
- 链接/导航辅助色：#2f6ea6

## 字体建议
- 中文正文：Noto Serif SC / Source Han Serif SC（后续可本地化）
- UI 与导航：Inter
- 若需统一自托管前，可先用系统字体回退

## 组件原则
- 卡片边框极轻
- 阴影极弱，更多用留白与分区建立层次
- 首页用问题卡片，正文页用双栏：主文 + 侧边索引
- 时间线、关系图、对照卡是重点组件

## 示意图策略
后续页面中要补充一组“解释型示意图”，它们不是装饰插画，而是帮助读者理解结构关系：

1. **时间轴图**
   - 首页：1911—1913 的总时间轴
   - 转折篇：宋案后的后果链时间轴

2. **权力关系图**
   - 制度篇：国会、内阁、总统、军权之间的关系示意
   - 人物篇：宋教仁、国民党、国会、袁世凯之间的关系图

3. **双路线对照图**
   - 宋教仁希望推动的路径：国会多数 → 责任内阁 → 政党政治 → 制度制衡
   - 民初现实路径：总统集权 → 军政压制 → 规则失效 → 暴力升级

4. **争议边界图**
   - 争议篇：稳定事实 / 主流解释 / 仍有争议 的三栏结构

## 图像生成与模型选择原则
后续如果需要为专题补充插图、关系图底图或风格化历史氛围图，优先走本地 grokapi 图像链路。

### 模型选择
不要把图像模型硬编码死。先请求本地 `http://127.0.0.1:8000/v1/models` 动态发现，再按用途选择：
- `grok-imagine-image-lite`：快速草图、示意底图
- `grok-imagine-image`：通用正式图
- `grok-imagine-image-pro`：质量优先、展示图
- `grok-imagine-image-edit`：后续做局部修改或二次编辑

### 使用原则
- 示意图优先服务“解释”，不追求花哨艺术感
- 历史专题避免影视海报风、革命宣传画风的过度渲染
- 关系图、流程图、对照图优先考虑 HTML/CSS/SVG 原生绘制；只有在需要气氛图或档案风插图时才调用图像模型

## 不做的事情
- 不把所有内容塞进单页
- 不把“谁是真凶”作为首页核心卖点
- 不使用密集数据看板式布局
