# 多页面信息架构

## 站点定位
多页面历史专题站，围绕“人物—事件—制度—转折—争议边界”递进展开。

## 一级导航
1. 首页 / 导览 (`index.html`)
2. 人物篇 (`pages/person.html`)
3. 事件篇 (`pages/case.html`)
4. 制度篇 (`pages/system.html`)
5. 转折篇 (`pages/turning-point.html`)
6. 争议篇 (`pages/debate.html`)

## 页面职责
### 首页
- 快速说明专题问题
- 提供总时间轴与四个入口
- 只讲“这专题为什么值得看”

### 人物篇
- 解释宋教仁为何关键
- 重点讲责任内阁与政党政治路线
- 不做冗长人物传记

### 事件篇
- 讲清 1912—1913 的关键链条
- 讲“宋案打断了什么”
- 案情只讲必要部分

### 制度篇（灵魂页）
- 解释民初共和为什么没有按共和方式运转
- 重点做四大制度困境
- 放对照卡与权力结构图

### 转折篇
- 讲宋案如何加速共和失衡
- 强调“转折点之一”而非“唯一原因”

### 争议篇
- 告诉读者哪些能确定、哪些需谨慎、哪些仍有争议
- 作为史料分寸与方法感的收束页面

## 共享组件
- 顶部主导航
- 页面内目录锚点
- 时间轴组件
- 对照卡组件
- 争议标签组件（稳定事实 / 主流解释 / 仍有争议）
- 页尾“继续阅读”跨页跳转

## grokapi 模型策略
后续如需插图或图像辅助，优先由本地 grokapi 动态发现模型，而不是手写固定模型名。

当前已验证本地 `http://127.0.0.1:8000/v1/models` 可返回：
- 文本模型：`grok-4.20-fast`、`grok-4.20-auto`、`grok-4.20-expert` 等
- 图像模型：`grok-imagine-image-lite`、`grok-imagine-image`、`grok-imagine-image-pro`、`grok-imagine-image-edit`

原则：
- 文本采料优先 `grok-4.20-fast` 起步
- 若后续需要更强整合/润色，可切到 `grok-4.20-auto` 或更高档位
- 图像先查模型列表，再按用途自动选型
